Forge API

Serverless lead intelligence API. Submit a location and a Blueprint product — get back hot, reachable prospects scored against that product, with sale strategies, confidence scores, a place_id, and a Google Maps link. The API discovers real businesses from Google Places (the only lead source) and holds all provider keys server-side; callers never send LLM, Places, or enrichment keys.

All processing runs in a single synchronous call. The pipeline executes: Google Places discovery → LLM scoring → format validation → contact enrichment (Hunter.io + AbstractAPI, server-side) → reachability verification. Only hot leads whose websites resolve are returned. To respect Google Maps Platform terms, responses contain place_id + a place_id Maps link + Curcible-generated intelligence — not cached Places content (name/address/phone).

Authentication

Every request must include a Bearer token in the Authorization header. API keys are issued by Curcible.

# Header
Authorization: Bearer YOUR_FORGE_API_KEY

Requests without a valid key return 401 Unauthorized. Keys are issued per tenant, hashed at rest, rate-limited, quota-metered, and can be revoked or rotated.

Important: Your API key authenticates access to the Forge API only. You do NOT send LLM, Places, or enrichment keys — these are held server-side by Curcible.

Base URL

https://YOUR_SITE.netlify.app/api/v1Base

All endpoints are relative to this base. Both the clean URL and the Netlify function URL are valid:

https://YOUR_SITE.netlify.app/api/v1/leads
# or directly:
https://YOUR_SITE.netlify.app/.netlify/functions/forge-leads

POST /leads

POST /api/v1/leads

Runs the full lead intelligence pipeline and returns qualified leads for the specified location and Blueprint product.

Headers

HeaderValue
AuthorizationBearer <FORGE_API_KEY> — required
Content-Typeapplication/json — required

Request body

FieldTypeReqDescription
statestringreqState or province to target. E.g. "California", "Ontario"
countrystringreqCountry. E.g. "United States", "Canada"
productstringreqBlueprint product ID. One of: P01–P08
llmProviderstringreqLLM provider to use for discovery. See Providers
llmModelstringreqModel ID within the chosen provider
llmApiKeystringreqYour personal API key for the chosen LLM provider
hunterApiKeystringoptHunter.io key for email discovery from business domains. Free: 25 searches/mo at hunter.io
abstractEmailKeystringoptAbstractAPI Email Validation key for SMTP verification. Free: 100/mo at abstractapi.com
abstractPhoneKeystringoptAbstractAPI Phone Validation key for carrier & line-type lookup. Free: 100/mo at abstractapi.com
Latency note: A typical request without enrichment keys completes in 4–8 seconds. With hunterApiKey the overhead is negligible. With AbstractAPI keys, add ~1.1s per check across the top 3 hot leads (email + phone = up to ~7 extra seconds).

Errors

All errors return { "ok": false, "error": "...", "details": {...} }.

StatusMeaning
200Success. leads[] may be empty if no hot + live leads passed all filters.
400Bad request. Missing or invalid fields. Check details.missing[].
401Unauthorized. API key missing or invalid.
405Method not allowed. Only POST is accepted on /leads.
500Pipeline error. LLM parse failure, network timeout, or provider outage. Check details.message.
{
  "ok":      false,
  "error":   "Missing required fields.",
  "details": { "missing": ["llmApiKey", "product"] }
}Error 400

JavaScript (fetch)

const response = await fetch("https://YOUR_SITE.netlify.app/api/v1/leads", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_FORGE_API_KEY",
    "Content-Type":  "application/json",
  },
  body: JSON.stringify({
    state:            "Texas",
    country:          "United States",
    product:          "P01",
    llmProvider:      "groq",
    llmModel:         "llama-3.3-70b-versatile",
    llmApiKey:        "gsk_YOUR_GROQ_KEY",
    hunterApiKey:     "YOUR_HUNTER_KEY",      // optional
    abstractEmailKey: "YOUR_ABSTRACT_EMAIL_KEY", // optional
    abstractPhoneKey: "YOUR_ABSTRACT_PHONE_KEY", // optional
  }),
})

const { ok, meta, leads } = await response.json()
console.log(`${meta.qualified} qualified leads for ${meta.productName}`)
leads.forEach(lead => console.log(lead.name, lead.sale_strategy))JS

Python (requests)

import requests

response = requests.post(
    "https://YOUR_SITE.netlify.app/api/v1/leads",
    headers={
        "Authorization": "Bearer YOUR_FORGE_API_KEY",
        "Content-Type":  "application/json",
    },
    json={
        "state":            "Texas",
        "country":          "United States",
        "product":          "P01",
        "llmProvider":      "groq",
        "llmModel":         "llama-3.3-70b-versatile",
        "llmApiKey":        "gsk_YOUR_GROQ_KEY",
        # optional enrichment keys:
        "hunterApiKey":     "YOUR_HUNTER_KEY",
        "abstractEmailKey": "YOUR_ABSTRACT_EMAIL_KEY",
        "abstractPhoneKey": "YOUR_ABSTRACT_PHONE_KEY",
    }
)

data = response.json()
print(f"{data['meta']['qualified']} leads returned")
for lead in data["leads"]:
    print(lead["name"], "—", lead["problem_solved"])Python
