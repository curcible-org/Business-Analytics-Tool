FAQ

Common questions.

Everything you need to know before purchasing.

What booking sources does SchedCore accept?

Web forms, Calendly (booking.created webhook), WhatsApp messages forwarded via n8n, and direct API calls. Any source that can send an HTTP request to a webhook URL can trigger SchedCore.

Does it work if my calendar is fully booked?

Yes. If no available slots are found in the 7-day window, SchedCore sends a "no availability" response to the requester with a prompt to contact you directly. The look-ahead window is configurable — extend it to 14 or 21 days for busier calendars.

Is the Twilio SMS required?

No. The SMS node is optional and fully independent. Disable it without affecting any other part of the workflow. The email confirmation and calendar event still fire — SMS is a convenience layer on top.

Can I customise the working hours and days?

Yes. Working hours (default 9am–5pm) and available days (default Mon–Fri) are set in the "Compute Available Slots" code node. Change these values once and they apply to every booking SchedCore handles from that point forward.

Does it work alongside Calendly or does it replace it?

Both modes work. You can use Calendly for the booking page and SchedCore for post-booking automation (confirmation email, SMS, HubSpot log, Slack notification). Or you can bypass Calendly entirely and use a web form directly connected to SchedCore.
