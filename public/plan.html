How This Plan Works

Each product is built start to finish before the next one begins. No context switching, no half-built workflows sitting idle. When a product is live and the client has signed off, you move to the next one.

The sequence is ordered by setup complexity and dependency chain — easiest products first. This means you're generating client revenue and building confidence early, while the more complex products (WhatsApp, LinkedIn) come after your process is solid.

Definition of "done" matters. Each product has a clear exit criteria — specific metrics you hit before moving forward. Don't skip this. A half-deployed workflow that breaks in Week 8 is harder to fix than one you stabilised properly in Week 2.

Infrastructure setup happens in Week 1 as a standalone task before any product build. It serves all 8 products and only needs to be done once.

Week 1 · Foundation
Infrastructure Setup
~8–12 hours · One time · Serves all 8 products
n8n Setup
1
Decide: Cloud vs Self-hosted. If you'll use P04 (SocialCore with Blotato community node), self-hosted is required. Railway or Render are easiest to deploy — free tiers available. n8n Cloud works for all other 7 products.
2
Provision your n8n instance. If self-hosted: deploy on Railway (5-minute deploy), set up SSL, configure your webhook base URL. If Cloud: confirm community nodes are enabled under Settings.
3
Install community nodes. In Admin Panel → Community Nodes → install n8n-nodes-blotato. Do this now even though P04 is Week 7.
4
Test a simple webhook. Create a test workflow with a webhook trigger. Confirm your n8n instance is publicly reachable (required for Zoom, WhatsApp, and email triggers).
Credentials & Accounts
5
OpenAI account + billing. Set a monthly spend limit ($50–100 to start). Every product except P04 uses OpenAI. One account, multiple workflows.
6
Google Workspace OAuth2. Create a Google Cloud project. Enable: Gmail API, Google Drive API, Google Calendar API, Sheets API, YouTube Data API v3. Create OAuth2 credentials. Store Client ID and Secret.
7
Qdrant cloud account. Sign up at qdrant.tech — free tier is enough to start. You'll use this for P01 (email KB) and P06 (WhatsApp KB). Create one cluster, note the URL and API key.
8
Credentials document. Create a private doc (Notion or 1Password) with all API keys, OAuth tokens, and account links. You will reference this constantly over 19 weeks.
⚠
Meta Developer App — submit today. Go to developers.facebook.com → Create App → Business → Add WhatsApp product → Submit for review. You won't use this until Week 11. But approval takes 7–14 days, and if you submit it in Week 10, you'll wait 2 weeks before you can touch P06. 30 minutes now saves 2 weeks later.
